# ROADMAP.md — ASBUH Portal → ASBUH.AUTOPILOT (Этап 2+)

## Принцип
После MVP не “наваливаем функций”, а включаем автопилот: сроки, задачи, процессы, интеграции.  
Если MVP — “кабина самолёта”, то Этап 2 — “приборная панель и автопилот”.

---

## Этап 2 — Рабочий контур задач и сроков (AUTOPILOT Core)
### 2.1 Задачи и календарь
- Генерация задач по параметрам организации (система Н/О, ЭЦП, канал сдачи отчётности, тип обслуживания).
- Повторяющиеся задачи, дедлайны, статусы, приоритеты, чек-листы.
- Дашборд “светофор” (красное/жёлтое/зелёное) для руководителя.
**Зависимости:** корректная модель организаций/участков/прав из MVP.

### 2.2 Уведомления и эскалации
- Email/Telegram уведомления по дедлайнам и статусам.
- Эскалация руководителю при просрочке.
- Логи отправок и повторов.

### 2.3 Расширение “карточки организации”
- UI-конфигуратор дополнительных полей (если в MVP сделали только структуру).
- Настройка “какие поля обязательны” по типу обслуживания/режиму.

---

## Этап 3 — Интеграции (там, где это даёт выгоду)
### 3.1 Провайдеры отчётности
- Контур/СБИС/Астрал: статусы сдачи, напоминания, импорт событий (если возможно).
**Зависимости:** задачи/календарь, единая модель периодов.

### 3.2 Банки и выписки
- Импорт выписок/событий (минимум: загрузка файлов + разметка; максимум: API).
**Зависимости:** безопасный контур секретов.

### 3.3 Коммуникации
- Шаблоны писем/сообщений клиентам, авто-запрос документов по задачам.
- История коммуникаций по организации.

---

## Этап 4 — Документы 2.0
- OCR/поиск по содержимому (если нужно).
- Автоматическая классификация документов (тип/период).
- Согласование документов (черновик → на проверке → утверждено).

---

## Этап 5 — Аналитика и управленческие отчёты
- Загрузка команды (задачи/сроки по участкам/сотрудникам).
- Финансы по тарифам, задолженности, маржинальность участка.
- “узкие места” процессов (где чаще всего просрочки).

---

## Этап 6 — Усиление безопасности
- 2FA (TOTP), политики доступа по устройствам/географии (при необходимости).
- Продвинутый аудит и алерты (подозрительная активность).
- Ротация ключей/секретов, периодический доступ (just-in-time).

---

## Технические “крючки”, которые нельзя сломать после MVP
- Multi-tenant изоляция данных.
- Аудит действий (минимум: кто/что/когда).
- Расширяемость карточки организации.
- Модульность: чтобы задачи/интеграции подключались без переписывания базы.
